# Claude Code Plan Mode & AskUserQuestion — Deep Analysis

## How Claude Code Does It

### Architecture: Tool-Based, Not Text-Parsing

Claude Code's approach is **fundamentally different** from our current implementation. Instead of parsing the agent's text output for questions, Claude Code uses a **dedicated tool** called `AskUserQuestion` that the agent explicitly calls with structured data.

**Our current approach**: Agent writes questions as markdown text → we regex-parse them → render a selector
**Claude Code's approach**: Agent calls `AskUserQuestion` tool with structured JSON → tool pauses execution → UI renders interactive prompt → answers returned as tool result

### The AskUserQuestion Tool

```typescript
interface AskUserQuestionInput {
  questions: Question[];  // 1-4 questions
  answers?: Record<string, string>;  // filled by UI, returned to agent
  metadata?: { source?: string };  // analytics tracking
}

interface Question {
  question: string;       // "Which library should we use for date formatting?"
  header: string;         // "Library" (max 12 chars, shown as chip/tag)
  options: Option[];      // 2-4 options
  multiSelect: boolean;   // false = radio, true = checkbox
}

interface Option {
  label: string;          // 1-5 words: "Day.js"
  description: string;    // "Lightweight, Moment.js-compatible API"
}
```

**Key constraints:**
- 1-4 questions per call
- 2-4 options per question
- NO "Other" option in the tool call — it's **auto-generated by the UI**
- `header` is a chip/tag label (max 12 chars)
- Option `label` is 1-5 words
- Option `description` explains trade-offs/implications
- `multiSelect` enables checkboxes instead of radio buttons

### UI Rendering

Claude Code renders questions using an Ink/React component with:

1. **Tab navigation bar** showing Q1 ☐, Q2 ☐, Q3 ☑ (checkbox indicators for answered)
2. **Arrow key selection** (↑/↓) to navigate options
3. **Enter** to select an option
4. **Auto-generated "Other" option** with inline text input
5. **Ctrl+G** to open external editor for longer "Other" text
6. **Submit button** at the end of the tab bar
7. For multi-select: checkboxes (☐/☑) instead of radio buttons

### Answer Format

Answers are returned to the agent as a tool result:
```
"User has answered your questions: \"Which library?\"=\"Day.js\", \"What scope?\"=\"Medium (3-5 files)\""
```

### Plan Mode Integration

- `AskUserQuestion` works **both inside and outside plan mode**
- In plan mode: used to "clarify requirements or choose between approaches BEFORE finalizing your plan"
- **Critical rule**: Do NOT use `AskUserQuestion` for plan approval — that's what `ExitPlanMode` is for
- Plan entry is via Shift+Tab toggle (like our Ctrl+P), not a tool

### ExitPlanMode Tool

```typescript
interface ExitPlanModeInput {
  allowedPrompts?: { tool: "Bash"; prompt: string }[];  // semantic permissions
  pushToRemote?: boolean;
  remoteSessionId?: string;
}
```

When exiting plan mode, the agent specifies **semantic permissions** (e.g., "run tests", "install dependencies") rather than specific commands. This is more flexible than whitelisting.

| Feature | Claude Code | ggcoder (current) |
|---|---|---|
| Question delivery | Dedicated tool with structured JSON | Text parsing with regex |
| "Other" option | Auto-generated by UI | Included in agent's options |
| Multi-select | ☐/☑ checkboxes | Not supported |

## Integration Plan for ggcoder

### Phase 1: Create `ask_user_question` Tool

**New file**: `src/tools/ask-user-question.ts`

Create a tool matching Claude Code's schema:
- Input: `questions` array with `question`, `header`, `options` (label + description), `multiSelect`
- The tool **does not execute** — it returns a special result that signals the UI to show the interactive prompt
- Uses a Promise-based pattern: tool creates a promise, UI resolves it when user answers

**Key design decision**: The tool needs to **pause the agent loop** and wait for user input. This requires:
1. The tool's `execute` function returns a Promise that doesn't resolve until the user answers
2. The UI subscribes to a "question pending" event and renders the selector
3. When the user answers, the promise resolves with the formatted answer
4. The agent loop continues with the tool result

### Phase 2: Build Interactive Question UI Component

**Replace**: `src/ui/components/PlanQuestionSelector.tsx`

3. **Auto "Other" option** with inline text editor
4. **Multi-select** support with checkboxes
5. **Arrow keys** for option navigation
6. **Tab/Shift+Tab** for question navigation
7. **Enter** to select/submit

### Phase 3: Wire Into Agent Loop

## Implementation Architecture

```
Agent calls ask_user_question tool
  ↓
Tool creates Promise + emits "question_pending" event
  ↓
App.tsx detects event → shows QuestionUI component
  ↓
User navigates options (↑↓), switches questions (Tab)
  ↓
User selects answer or types "Other" text
  ↓
User hits Submit → answers collected
  ↓
Promise resolves with formatted answer string
  ↓
Agent receives tool_result and continues
```

### Event Bus Pattern

Use the existing `event-bus.ts` (or create a simple one) to communicate between the tool and the UI:

```typescript
// In ask-user-question.ts
eventBus.emit("question_pending", { questions, resolve });

// In App.tsx
eventBus.on("question_pending", ({ questions, resolve }) => {
  setQuestionState({ questions, resolve });
});
```
